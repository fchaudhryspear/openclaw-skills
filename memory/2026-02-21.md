## Snowflake Integration - Portal Fix (Feb 21, 2026)

### Problem
Portal Snowflake tab showing "Disconnected" / 401 errors on `/snowflake/health` endpoint.

### Root Causes Found
1. Lambda code only supported key-pair auth (not password)
2. Snowflake database was `APPLICATIONS` (doesn't exist) - should be `LENDING_DB`
3. Powertools logger correlation_id error blocking Lambda execution

### Fixes Applied

**Lambda Code Updates** (`data-lake-monitoring/functions/monitoring_api/lambda_function.py`):
- Added `SNOWFLAKE_PASSWORD` env var support
- Updated `get_snowflake_connection()` to try password auth first, then key-pair
- Disabled powertools correlation_id to fix `InvalidEnvelopeExpressionError`

**Lambda Environment Variables**:
```
SNOWFLAKE_ACCOUNT=SBUBJHD-TY50972
SNOWFLAKE_DATABASE=LENDING_DB
SNOWFLAKE_USER=lambda_api_user
SNOWFLAKE_PASSWORD=TempPass123!
SNOWFLAKE_WAREHOUSE=COMPUTE_WH
SNOWFLAKE_SCHEMA=PUBLIC
SNOWFLAKE_ROLE=ACCOUNTADMIN
TABLE_NAME=real-time-data-lake-loan-applications
```

**Snowflake Setup Required** (run as ACCOUNTADMIN):
```sql
CREATE USER IF NOT EXISTS lambda_api_user 
  PASSWORD = 'TempPass123!' 
  DEFAULT_ROLE = LENDING_DB_LAMBDA
  DEFAULT_WAREHOUSE = COMPUTE_WH
  MUST_CHANGE_PASSWORD = FALSE;

GRANT ROLE LENDING_DB_LAMBDA TO USER lambda_api_user;
GRANT USAGE ON DATABASE LENDING_DB TO ROLE LENDING_DB_LAMBDA;
GRANT USAGE ON SCHEMA LENDING_DB.PUBLIC TO ROLE LENDING_DB_LAMBDA;
GRANT SELECT ON ALL TABLES IN SCHEMA LENDING_DB.PUBLIC TO ROLE LENDING_DB_LAMBDA;
GRANT SELECT ON FUTURE TABLES IN SCHEMA LENDING_DB.PUBLIC TO ROLE LENDING_DB_LAMBDA;
```
