# Telegram Group Authorization Troubleshooting

- **Lesson Learned:** OpenClaw's Telegram integration requires specific `groupPolicy` and `dmPolicy` settings for group commands to work. The `groupPolicy` should be `"open"` (not `"any"` or `"allowlist"` unless `allowFrom` is `"*"`). Critically, **Telegram's Bot API privacy mode** must be toggled to "Group Privacy OFF" in BotFather for the bot to see all messages in groups and respond to commands without explicit mentions.
- **Resolution Steps:**
    1. Ensure `groupPolicy: "open"` for all relevant Telegram groups in `openclaw.json`.
    2. Set `dmPolicy: "open"` in `channels.telegram` section if direct messages should also be open.
    3. Ensure `allowFrom` includes `"*"` if `dmPolicy` or `groupPolicy` is `"open"`.
    4. **Crucial:** Toggle "Group Privacy OFF" for the bot in BotFather.
    5. Perform a full OpenClaw restart after any configuration changes (`pm2 stop all && pm2 kill && pm2 start openclaw && pm2 save`).
