# 2026-02-24 - IRR Analysis Tool Project Update

## Project: IRR Analysis Tool (from fchaudhryspear/IRR_Quoting_Tool)
- User wants to migrate the Streamlit application from Replit to AWS App Runner.
- Current status: Codebase has been modularized and prepared for AWS deployment.

### Key Changes Made:
1.  **Code Modularization:**
    *   `app.py`: Now primarily handles Streamlit UI, importing functions from other modules.
    *   `data_loaders.py`: Created to house all hardcoded financial curves, SOFR fetching logic, and Excel parsing functions.
    *   `financial_logic.py`: Created to encapsulate the core `run_model` for financial calculations.
    *   `database_manager.py`: Created to manage all interactions with the database for scenario saving/loading.
    *   `chart_generator.py`: Created to hold the `create_charts` function for Plotly visualizations.

2.  **Database Migration (Code Side):**
    *   `database_manager.py` was updated to use `psycopg2` for PostgreSQL instead of `sqlite3`.
    *   Database connection details are now fetched from AWS Secrets Manager using `boto3`, with environment variable fallbacks for local development.

3.  **Dockerization:**
    *   `Dockerfile`: Created to define the Docker image build process for the Streamlit application. Includes system dependencies for `psycopg2` and runs the Streamlit app.
    *   `requirements.txt`: Updated to include all necessary Python packages, including `psycopg2-binary` and `boto3`.

### Next Steps (Deployment to AWS App Runner - User's Manual Actions Required):
- User needs to push the updated code to their GitHub repository (`fchaudhryspear/IRR_Quoting_Tool`).
- User needs to provision an AWS RDS PostgreSQL instance.
- User needs to create a secret in AWS Secrets Manager named `/streamlit/irr_tool/db_credentials` containing PostgreSQL credentials.
- User needs to create an IAM role for App Runner with permissions to read from Secrets Manager.
- User will then create an AWS App Runner service, linking it to the GitHub repository and configuring it with the correct port (8501) and IAM role.
